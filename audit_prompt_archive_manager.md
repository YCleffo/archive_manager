# Audit Prompt для проекта «Менеджер файлов / Архиватор»

Ты — senior Python/PySide6 security & performance reviewer. Проведи строгий аудит десктопного приложения-файлового менеджера/архиватора.

## Контекст
Проект: Python 3.10+, PySide6/Qt, файловый менеджер с поиском, созданием ZIP, распаковкой ZIP/TAR, копированием, перемещением, удалением, переименованием, подсчётом размера папок и GUI.

## Что проверить
1. **Безопасность архивов**
   - Zip Slip / Tar Slip.
   - Символические и hard links в TAR.
   - Абсолютные пути, `..`, UNC/сетевые пути, Windows drive paths.
   - Перезапись существующих файлов при распаковке.
   - Zip/Tar bombs: лимит количества файлов, суммарного размера, размера одного файла, глубины путей.
   - Архивация архива внутрь самого себя, если выходной ZIP лежит внутри выбранной папки.

2. **Файловые операции**
   - Безопасность удаления: корзина vs безвозвратное удаление.
   - Копирование/перемещение папки внутрь самой себя.
   - Обработка symlink/junction/reparse points.
   - Race condition между проверкой пути и действием.
   - Ошибки доступа, сетевые папки, длинные пути, занятые файлы.

3. **GUI/потоки**
   - Не блокируется ли UI при больших папках.
   - Можно ли отменить долгие операции.
   - Нет ли гонок между старым и новым поиском.
   - Корректно ли закрывается приложение без `os._exit()` и без повреждения файлов.

4. **Производительность**
   - `rglob`, `stat`, чтение содержимого файлов, подсчёт размеров папок.
   - Кэширование размеров, ленивые списки, batch-вставка строк в таблицу.
   - Ограничение результатов поиска, исключение системных/служебных папок.

5. **Кроссплатформенность**
   - Windows/Linux/macOS пути.
   - Разделители `/` и `\\`.
   - `open`, `xdg-open`, `os.startfile`.
   - Документация и наличие `run.sh`, если он указан в README.

6. **Качество кода**
   - Повторяющиеся методы/делегаты UI.
   - Типизация, ruff/mypy, pytest.
   - Разделение UI и бизнес-логики.
   - Логирование вместо скрытого `except: pass`.
